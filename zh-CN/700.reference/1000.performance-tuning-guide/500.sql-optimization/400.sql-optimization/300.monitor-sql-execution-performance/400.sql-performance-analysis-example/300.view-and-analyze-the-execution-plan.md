# 查看执行计划形状并做分析

根据如下步骤查看执行计划形状并做分析：

1. 通过 `EXPLAIN` 命令查看执行计划形状，或者通过实时执行计划展示查看缓存执行计划。详细信息请参见 [SQL 执行计划简介](../../../200.sql-execution-plan/100.introduction-to-sql-execution-plans.md)和 [实时执行计划展示](../../../200.sql-execution-plan/500.real-time-execution-plan-display.md)。

2. 获得执行计划形状后，可以分析索引、联接顺序、 联接算法等选择是否合理正确。详细信息请参见 [访问路径](../../600.query-optimization/100.access-path/100.access-path-overview.md)、[联接顺序](../../600.query-optimization/200.join-algorithm/400.join-order.md)和 [联接算法](../../600.query-optimization/200.join-algorithm/200.join-algorithm.md)。
